# Public site structure, blog workflow, and dev tooling content plan

Source package: 20260702-205728-10479807-context

Duration: 28:29

## Summary

The memo outlines a plan for a public website organized around experience, ongoing projects, blog posts, and reference material. It emphasizes using voice-over transcripts to generate accurate project updates and blog content, refining tone and voice, and publishing supporting material for dev tools, skills, and related artifacts. It also considers how to separate Twitter from the website while using tweets mainly to point back to the blog or engage with others.

## Key Points

- Public site should clearly present experience, ongoing projects, and blog-style updates.
- Projects are envisioned as containers with timelines, milestones, images, and post-based updates.
- Voice-over/transcript-driven blog posts are seen as a strong way to capture work context, learnings, and plans.
- Blog tone, voice, imagery, code rendering, and OG thumbnails need refinement.
- Experience section should include a mix of programming and non-programming work without diluting the resume.
- Some project posts may need visibility flags for private or unreleased work.
- There is a potential reference/skills section for explainers, annotated skills, and publishing reusable skills.
- Twitter is considered separate from the site, mainly for engagement and linking back to the blog.
- Several dev tools and visualization projects should be written up, published, and improved.
- A recurring voice-over feedback loop could help document what works, what doesn’t, and what to change.
